<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User-initiated inject request → live process | The most sensitive boundary: toolkit writes to a live process and loads code into it |
| agentDllPath (string from renderer) → CreateRemoteThread(LoadLibraryA) | The DLL path must be trusted — if an attacker can write to the install dir and place a file with the agent name, it would be loaded |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | attach to wrong process | mitigate | GetFileVersionInfo ProductName check runs BEFORE inject in both paths; wrong ProductName = error + no inject |
| T-03-02 | Tampering | agent DLL path trust | mitigate | agentDllPath is constructed by the host from an app-install-relative path (not user-supplied freeform); document in SUMMARY the path construction; attacker would need write access to the install directory |
| T-03-05 | Elevation of Privilege | OpenProcess ACCESS_DENIED | mitigate | Return structured error "file-patch mode" reason — NO AdjustTokenPrivileges, NO UAC manifest escalation request; renderer degrades gracefully (D-08) |
| T-03-06 | Tampering | write before verify | mitigate | Phase 3 has no write path (LIVE-03 is Phase 5); the inject only starts the read-verify agent; the 4-sentinel gate in agent_main prevents any write until Phase 5 adds it |
| T-03-04 | Info-disclosure | VirtualQuery-unguarded RPM reads | mitigate | ReadProcessRegion has a 4096-byte max and validates ReadProcessMemory return; agent-side transforms are validated by sentinels before channelWrite |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages; N/A |
</threat_model>
