<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Renderer UI → host addon IPC | The UI triggers attach/inject; must not allow arbitrary PID or DLL path injection from a compromised renderer |
| Live memory view → renderer | Raw RPM bytes displayed in HexInspector — untrusted bytes from client memory; no eval, no deserialization |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Elevation of Privilege | attach error → file-patch degrade | mitigate | attachError() sets mode='file-patch' and shows reason in disabled state; no auto-escalation path in the UI (D-08) |
| T-03-04 | Info-disclosure | raw HexInspector over RPM bytes | accept | HexInspector renders raw bytes as hex display only — no deserialization, no eval, no pointer dereference in the renderer; accepted because the user explicitly requested the raw view |
| T-03-06 | Tampering | write gate UI surface | mitigate | Phase 3 HUD is READ-ONLY — no write UI is present; allSentinelsPassed is for future Phase-5 write gating; no write button in this panel |
| T-03-01 | Spoofing | PID picker shows SWG clients | accept | ListSWGClientPids filters by ExeName — reduces the wrong-target risk; the identity check in inject_binding is the authoritative gate |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages; N/A |
</threat_model>
