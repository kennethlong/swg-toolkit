    Export async function gitCommit(repoPath: string, message: string, stagePaths: string[]): Promise<string>:
    Guard: if (stagePaths.length === 0) throw new Error('No paths to stage — never use git add . (D-04-15)').
    Guard: message must not be empty (throw 'Commit message required').
    Sanitize message: strip null bytes and control chars (message.replace(/[\x00-\x1F\x7F]/g, ' ').trim()); cap at 500 chars. This prevents injection via message content.
    useVcsStore.getState().beginCommit().
    Stage explicitly: await execFileAsync('git', ['add', '--', ...stagePaths], {cwd: repoPath}).

    Return shortSha.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| commit message textarea → execFile git commit | User-typed commit message flows into git argv; must never allow shell metacharacters to escape argument isolation |
| staged file path list → execFile git add | File paths in stagePaths flow into git argv; must be relative workspace-relative paths |
| pre-commit hook output → UI | Hook stderr is displayed to user; must not expose sensitive paths |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-16 | Tampering | gitCommit message → execFile argv | mitigate | execFile isolation means argv[N] is passed as a literal argument, NOT shell-interpolated; even so, strip null bytes and control chars from message before passing (defense-in-depth); never use exec() |
| T-04-17 | Tampering | stagePaths containing '../' escape | mitigate | Each path in stagePaths must be relative (not absolute, not starting with '../') before passing to git add; validate in gitCommit before the execFileAsync call |
| T-04-18 | Information Disclosure | copyright leak — retail bytes committed | mitigate | Defense-in-depth D-04-15: .gitignore *.tre (workspaceService), explicit-path staging (never git add .), pre-commit hook (workspaceService installs), app-side getGuardStatus check before every commit |
| T-04-19 | Denial of Service | large binary committed and bloating clone | mitigate | Pre-commit hook size guard (> 50 MB → reject); app-side getGuardStatus also checks size |
| T-04-SC | Tampering | npm/pip installs | mitigate | No new npm packages; slopcheck not required |
</threat_model>
