<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-dragged file path → staging entry virtualPath | Dragged filenames and user-typed virtual paths may contain path traversal sequences |
| staging list → packPatch call | The replacement file path in a StagingEntry is used directly in packPatch; must not escape the workspace |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Tampering | StagingPanel virtualPath input | mitigate | Reject any virtualPath containing '..' or an absolute path separator ('\' or '/' at position 0) before calling addEntry; show inline ✕ error per UI-SPEC |
| T-04-06 | Tampering | StagingPanel drag-drop replacement file path | mitigate | The replacementFilePath is only ever READ (passed to packPatch which reads it); validate it is absolute and within the workspace or a child dir before adding to staging |
| T-04-07 | Information Disclosure | 'source missing' check via fs.existsSync | accept | existsSync on a local path does not leak sensitive info; the result is shown only in the user's own UI |
| T-04-SC | Tampering | npm/pip installs | mitigate | No new npm packages installed; slopcheck not required |
</threat_model>
