## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input → workspace path | User-chosen folder path; may be attacker-controlled if the toolkit is used programmatically or from IPC |
| workspaceService → disk | Service writes .gitignore, .gitattributes, manifest.json, and pre-commit hook into user-chosen dir |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | workspaceService createWorkspace path | mitigate | Validate folderPath is absolute and non-null-byte before any fs operation; normalize with path.normalize and check the result; refuse paths resolving to Windows system roots (C:\Windows, C:\Program Files) |
| T-04-02 | Tampering | git init via execFile | mitigate | Use execFile('git', ['init', '-b', 'main'], {cwd: folderPath}) — arg array, no interpolation; catch ENOENT and warn (do not expose raw error to UI without sanitizing) |
| T-04-03 | Information Disclosure | .gitignore/pre-commit written to workspace | accept | User chose the folder; the files we write are defense-in-depth safeguards, not secrets; no PII written |
| T-04-04 | Denial of Service | mkdirSync on a very deep or UNC path | mitigate | Check path.isAbsolute(normalized); impose a max path length guard (< 200 chars on Windows) before mkdirSync |
| T-04-SC | Tampering | npm/pip installs | mitigate | This plan installs NO new npm packages; slopcheck not required |
</threat_model>
