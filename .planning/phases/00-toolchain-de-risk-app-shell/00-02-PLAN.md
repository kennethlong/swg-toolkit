<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| C++ addon → Node.js process | Native code runs with full process privileges; a crash takes down the utility process (by design — isolated from main/renderer) |
| cmake-js build → MSVC toolchain | C++ compiled by local MSVC; no remote code execution; build artifacts (.node) are local |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-00-03 | Tampering | native addon loaded in renderer | mitigate | Addon NEVER loaded in renderer (sandboxed; require() unavailable); loaded only in utility process (Plan 03). This plan's unit tests run in Node directly — not in the Electron renderer. Verified in Plan 03 E2E test (preload security spec). |
| T-00-04 | Denial of Service | Napi::SharedArrayBuffer::New with unchecked byteLength | mitigate | sab.cpp validates info.Length() >= 1 before accessing info[0]; zero-length SAB is allowed and tested (Test 6); excessively large byteLength will throw OOM in native — acceptable for Phase 0 (Phase 1 adds bounds validation for production use). |
| T-00-05 | Tampering | NAPI_EXPERIMENTAL flag accidentally enabling unstable API | accept | node-addon-api 8 does NOT require NAPI_EXPERIMENTAL for SharedArrayBuffer::New; compile definition is NAPI_VERSION=8 only. Acceptance criterion verifies no NAPI_EXPERIMENTAL in CMakeLists.txt. |
| T-00-SC | Tampering | cmake-js / node-addon-api npm packages | mitigate | Both packages verified [ASSUMED OK] in RESEARCH.md §Package Legitimacy Audit; cmake-js 11yr history, node-addon-api 9yr history, Microsoft/nodejs orgs respectively. No checkpoint required. |
</threat_model>
