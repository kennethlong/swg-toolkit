<objective>
Build the minimal cmake-js C++ Node-API addon for Phase 0: two exports (`hello()` → "pong" and `allocateSab(byteLength)` → SharedArrayBuffer), prove the addon with Vitest unit tests, then produce the prebuildify/node-gyp-build no-compiler distribution path that FND-02 explicitly requires — and prove it NON-CIRCULARLY (build/ moved aside, loaded path asserted to be prebuilds/).

Purpose: Proves the C++ → N-API toolchain works (cmake-js, MSVC v143/v145, node-addon-api ^8.8.0 with NAPI_EXPERIMENTAL) and produces the `.node` binary that Plan 03 loads in the utility process. The de-anchoring review found that `Napi::SharedArrayBuffer` is **experimental-gated** — the build MUST define `NAPI_EXPERIMENTAL` and pin `node-addon-api@^8.8.0`. FND-02 also requires "prebuildify distribution" (no-compiler distribution) — one of the two things this phase exists to de-risk. Round 2 found the prior prebuildify proof was CIRCULAR (review fix HIGH-1 / Sonnet + Opus): prebuilds/ was built with MSVC present and resolved on the same box with build/Release co-resident, so a green test could not prove the load came from prebuilds/ or that no compiler was needed. This plan closes that hole: after the cmake-js build, it places the emitted `.node` into the `prebuilds/<platform>-<arch>/` layout (a small `prebuild` script step — node-gyp-build stays the single resolver), moves build/ aside, re-runs the suite so resolution can ONLY go through prebuilds/, and asserts the loaded .node path is the prebuilds/ one. Because the addon is pure N-API (`NODE_API_MODULE` + `NAPI_VERSION=8` + `--napi`), that ONE prebuild is ABI-stable across Node AND Electron — the same binary that vitest loads also loads under Electron, so there is no separate Electron-ABI build (round-3 / Cursor CUR-1). The packaged-Electron RUNTIME LOAD is proven in Plan 05.

Output: `swg_native_core.node` built by cmake-js; a `prebuilds/` layout produced by prebuildify and resolved by node-gyp-build via `index.js`; a non-circular resolution proof; TypeScript type declarations (`index.d.ts`); passing unit tests for both exports. CMakeLists.txt is structured for Phase 1 expansion but contains NO IFF/TRE C++ (per CONTEXT.md Deferred).
</objective>
