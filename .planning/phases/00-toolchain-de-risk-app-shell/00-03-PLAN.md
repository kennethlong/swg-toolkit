<objective>
Wire the Electron security posture and the complete utility-process → SAB → renderer pipeline: main process creates BrowserWindow with contextIsolation, registers COOP/COEP response headers synchronously before loadURL, forks the utility process, and relays the SharedArrayBuffer to the renderer via MessageChannelMain. The preload exposes a narrow typed bridge. The renderer observes crossOriginIsolated === true and reads the C++-written 0xDEAD sentinel from the SAB.

Purpose: This plan proves success criteria 1 (security posture), 2 (hello round-trip), and 3 (SAB round-trip / cross-origin isolation) simultaneously, which is D-04 from CONTEXT.md. These three criteria cannot be proven independently — they share the same IPC path.

Output: Three source files (main.ts, preload.ts, utility-worker.ts) that together deliver a running Electron app with the zero-copy SAB channel live and the security posture locked.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| renderer → preload bridge (window.api) | Untrusted renderer JS crosses here; must be a typed allowlist, not a passthrough |
| main process → utility process (MessagePort) | Trusted internal; utility process can crash without taking down the window |
| COOP/COEP headers → renderer isolation | Security boundary for SharedArrayBuffer access; must be set on every response |
