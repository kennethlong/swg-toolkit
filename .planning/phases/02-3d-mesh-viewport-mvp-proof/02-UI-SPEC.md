# Phase 2 — UI Design Contract: 3D Mesh Viewport (MVP Proof)

> Visual and interaction contract for the Phase 2 user-facing surfaces. Generated by gsd-ui-researcher.
> Consumed by gsd-planner (task decomposition), gsd-executor (implementation), gsd-ui-checker (validation), gsd-ui-auditor (retro audit).

**Scope (the new viewport-studio surfaces this phase ships into the locked Phase-0 dockview shell):**
1. **Live 3D viewport** — a real R3F/Three.js canvas replacing the Phase-0 viewport placeholder, with an orbit camera, lighting rig, grid, gizmo, render-mode chips, and a real-time stats overlay (VIEW-01). Geometry crosses the bridge zero-copy into `BufferGeometry`.
2. **LOD picker** — a user-selectable level-of-detail control over the resolved external LOD graph (`.lmg`/`.ldt`), D-02.
3. **Customization panel** — live, interactive color/palette pickers that re-tint the asset in real time (the explicit "beats TRE Explorer" wow moment), D-06.
4. **Material / texture inspector** — surfaces the resolved multi-map material (diffuse/normal/spec/env/effect) and `.dds`/`.pal` provenance, D-07.
5. **Animation transport** — `.ans` picker + timeline scrubber + play/pause/loop/speed for skeletal playback, plus a skeleton-helper toggle (VIEW-03, D-08).
6. **Appearance / missing-dependency panel** — the composed-appearance resolution result (which `.sat`→`.mgn`→`.skt`→`.sht`→`.dds`/`.pal` resolved) and a `missing: X` warning surface with placeholder substitution (D-03/D-04/D-05).
7. **Extract & export affordances** — extract a raw asset; export the viewed (rigged + animated) mesh to glTF / COLLADA (VIEW-04, D-10).

**Out of scope (do NOT design here — inherited locks, or other phases):**
- The app shell, titlebar, theme picker, panel chrome, docking, status bar — **all LOCKED by sketch 001 + `00-UI-SPEC.md`**. This spec consumes those tokens; it never redefines them.
- The TRE VFS browser, IFF structure tree, hex inspector — **shipped and locked in `01-UI-SPEC.md`**; Phase 2 consumes them as the asset-open entry point (open a file in the VFS → render it here).
- In-viewport mesh / UV / weight / rig **editing** — out of scope project-wide (bridge to Blender). Phase 2 is view + export only (CONTEXT Deferred).
- Re-import / round-trip of glTF/COLLADA back into SWG formats — Phase 2 export is one-way (D-10).
- Typed datatable / STF editors (Phase 5); live-injection gizmo drag into the running client (Phase 5); other appearance types — particles/terrain/portals (Phase 7).

**Inheritance:** Every token, accessibility rule, copy rule, and registry decision below is inherited verbatim from `00-UI-SPEC.md` and extended by `01-UI-SPEC.md`, unless a Phase-2 addition is explicitly called out. Where this spec adds rows (new icon-only controls, new copy strings, new overlay states) they EXTEND the prior exhaustive lists — they do not replace them.

## Spacing Scale — 3px DCC base grid (project grid exception, inherited)

This project deliberately overrides the standard 4px/8px web grid with a **3px DCC base unit** to match
Blender-style panel density (validated in sketch 001, locked in `00-UI-SPEC.md`). All spacing tokens are
multiples of the 3px project base unit. This is a documented, intentional project-level exception — the
gsd-ui-checker must evaluate spacing against this 3px grid, NOT a 4px or 8px grid.

Use these tokens exclusively. Do not use arbitrary pixel values (3D-canvas internals like camera offsets
and gizmo geometry are exempt — they are scene-space, not layout-space).

| Token | Value | Phase-2 usage |
|-------|-------|---------------|
| `--space-1` | 3px | Chip internal padding, transport-button gaps, color-swatch grid gap, slider-tick gap |
| `--space-2` | 6px | Viewport overlay chip clusters, LOD-picker item padding, customization-row gap, transport bar item spacing |
| `--space-3` | 9px | Customization section head padding, material-row vertical padding, missing-deps row padding |
| `--space-4` | 12px | Side-panel body padding, transport-bar horizontal inset, export-dialog body inset |
| `--space-6` | 18px | Inter-section spacing (between LOD picker / customization / material blocks in the Inspector) |
| `--space-8` | 24px | (reserved — major layout gaps) |

**Exceptions (inherited chrome dimensions / structural values — not spacing values):**
- `--tabstrip-h: 30px`, `--statusbar-h: 22px`, `--activity-w: 46px` — fixed inherited chrome.
- Viewport overlay chip insets: chips sit `8px` from the viewport edges and `calc(var(--tabstrip-h) + 8px)` from the top — **inherited verbatim from the Phase-0 `ViewportPanel.tsx`** (the existing `top/left/right: 8px`, `gap: 4px` overlay layout is the established, shipped pattern; preserve it). The 8px/4px overlay offsets are an existing structural exception, not new.
- Gizmo: 48×48px (inherited), bottom-right, opacity 0.8.
- Transport bar height: **30px** (= `--tabstrip-h`) for visual rhythm with the panel head.
- Color swatch tile: 18px × 18px (= `--space-6`) for a comfortable DCC-density pick target inside the customization grid.

## Typography

Inherited from `00-UI-SPEC.md` — two font families, four sizes, weights 400 / 600 only. Monospace is
**mandatory** for all numeric / technical readouts (this phase is metrics-heavy: fps, vert/tri counts,
frame numbers, bone counts, byte sizes, palette indices, channel values).

| Role | Token | Size | Weight | Line Height | Font | Phase-2 usage |
|------|-------|------|--------|-------------|------|---------------|
| Micro / technical | `--text-xs` | 11px | 400 | 1.5 | **mono** | Stats overlay (fps / verts / tris / draw calls), frame counter (`{cur}/{total}`), playback speed (`1.0×`), LOD distances, bone count, palette index, RGBA channel values, texture dimensions, file sizes |
| Label / control | `--text-sm` | 12px | 400 | 1.4 | sans | Chip labels (Solid/Wire/Textured), LOD picker option labels, transport control labels, customization-variable names, material-slot names, tab labels |
| Body / field | `--text-base` | 13px | 400 | 1.5 | sans | Section body copy, empty-state body, warning/error body, export-dialog prose |
| Subheading / name | `--text-md` | 14px | 600 | 1.3 | sans | Loaded-asset name, section heads ("Customization", "Animation", "Material", "Appearance"), missing-deps heading, error heading |

**Phase-2 rules (extend the inherited mono mandate):**
- **Monospace is mandatory** for: the viewport stats overlay (`persp · {W}×{H} · {fps} fps · {verts} v · {tris} t · {draws} dc · SAB ✓`), the animation frame/time counter (`{frame}/{total} · {t}s`), playback speed multiplier, LOD level + distance readouts, bone counts, customization palette indices and packed color values (`#AARRGGBB`), DDS format tags (`DXT1`/`DXT5`), and texture/mip dimensions.
- **4-character SWG format / chunk tags** shown inline (e.g. `SKMG`, `SMAT`, `CKAT`, `SSHT`) render in **mono** — they are literal 4-byte tokens, not prose (inherited from `01-UI-SPEC.md`).
- Sans for prose: section heads, control labels, empty-state copy, warning/error sentences, export dialog copy.

## Color

Inherited 60/30/10 split from `00-UI-SPEC.md`. Theme-driven; cyan is the active default and re-styles
automatically across all five themes (incl. high-contrast). Accent stays scarce — it marks
selection/active/success only, never decoration. State is **never** conveyed by color alone
(Accessibility Rule 1).

> **Critical distinction for this phase:** the SWG **asset's own customization colors** (rendered on the
> mesh via the customization palette, D-06) are *scene content*, NOT UI chrome. They are driven by the
> `.pal` palette + shader uniform and may be any color the artist authored. The 60/30/10 + accent-scarcity
> rules govern the **viewport UI chrome** (chips, panels, transport, pickers) — they do NOT constrain the
> colors painted onto the 3D model or shown inside a color-swatch picker. The gsd-ui-checker must evaluate
> color discipline on the chrome, and treat swatch tiles + the rendered model as exempt scene content.

| Role | Hex (default/cyan theme) | Token | Phase-2 usage |
|------|--------------------------|-------|---------------|
| Dominant (60%) — deep background | `#181818` | `--color-bg` | Shell frame, side-panel gaps |
| Viewport canvas background | radial `#2a2e26 → #141414` | (inherited Phase-0 viewport gradient) | The 3D canvas background; preserve the existing radial gradient + masked 28px dot grid as the empty/neutral backdrop |
| Secondary (30%) — panel surfaces | `#242424` body / `#323232` header | `--color-surface` / `--color-header` | Inspector side-panel bodies (LOD/customization/material/appearance), transport bar, export dialog |
| Nested rows / hover | `#2b2b2b` | `--color-surface-2` | Customization row hover, material-slot row hover, LOD option hover, missing-deps row hover |
| Accent (10%) — hologram cyan | `#38bec9` | `--color-accent` | See reserved-for list below |
| Success | `#38bec9` (accent) | `--color-accent` | Round-trip PASS for a parsed format, "resolved ✓", export-complete ✓, "SAB ✓" / "zero-copy ✓" in the stats overlay |
| Warning | `#e0a13a` | `--color-warn` | `missing: X` dependency warnings, placeholder-substitution notices, unsupported-anim-version notice (KFAT 0002), weight-truncation note |
| Destructive / error | `#e0584f` | `--color-danger` | Parse error, render failure, export failure, S3TC-unavailable hard error |
| Info | `#4a8cff` | `--color-info` | Neutral informational notes (e.g. "compressor: DXT1", "CKAT 0001", LOD distance ranges) |

**Accent reserved for (Phase-2 additions to the Phase-0/1 exhaustive lists — add nothing else):**
1. Active render-mode chip fill (Solid/Wire/Textured) + active camera-mode chip — **inherited verbatim** from the Phase-0 `ViewportPanel.tsx` chip styling (`background: --color-accent; color: --color-accent-text`).
2. Selected LOD level in the LOD picker (selected option: 2px accent left border + `--color-accent-dim` bg — inherited selection pattern).
3. Active animation-transport play state pip (the play/pause toggle shows accent when playing) + the scrubber playhead fill (`--color-accent` on a `--color-widget` track).
4. Focus ring on all interactive controls (`--focus-ring`) — inherited.
5. Format round-trip PASS glyph + "byte-exact ✓" caption in the material/appearance provenance readout (the SC-5 standing gate surfaced read-only — inherited from `01-UI-SPEC.md`).
6. "resolved ✓" win-pip on a successfully-resolved appearance dependency (the resolver-success indicator).
7. Skeleton-helper-active and bounding-box-active toggle state (chip active fill, inherited chip pattern).
8. Export-complete confirmation glyph (`✓ exported …`).

**Accent must NOT appear on:** the 3D model surface (that is scene content), inactive chips, unselected LOD levels, decorative panel borders, the grid, the gizmo (the gizmo uses its own fixed axis colors — Y green / X red / Z blue, inherited), disabled controls, or any non-interactive background. Semantic states (warn/error/info) use their OWN semantic token, never accent.

### Semantic-state color is always paired with a glyph (Accessibility Rule 1 extension)

Render / resolve / verify status is conveyed by **glyph + color together**, never color alone:

| State | Glyph | Color token | Label (mono/sans, `--text-xs`/`--text-sm`) |
|-------|-------|-------------|---------------------------------------------|
| Format round-trip PASS (SC-5, read-only) | `✓` | `--color-accent` | `byte-exact ✓` |
| Appearance dependency resolved | `✓` | `--color-accent` | `resolved ✓` |
| Dependency missing (placeholder substituted) | `⚠` | `--color-warn` | `missing: {name} — placeholder` |
| Unsupported animation version (KFAT 0002) | `≈` | `--color-warn` | `unsupported legacy animation (KFAT 0002)` |
| Skin-weight truncation (>X% weight lost on a vertex) | `▴` | `--color-warn` | `{n} verts: weights truncated to 4` |
| Parse / render error | `✕` | `--color-danger` | `parse error` / `render failed` |
| S3TC unavailable (DXT GPU path) | `✕` | `--color-danger` | `S3TC unavailable — using CPU decode` (note: warn if CPU fallback succeeds, danger if it cannot) |
| Export complete | `✓` | `--color-accent` | `exported {file}` |
| Zero-copy verified (CORE-06) | `✓` | `--color-accent` | `zero-copy ✓` (stats overlay) |

## Surface 1 — Live 3D Viewport (CENTER / Viewport panel)

Replaces the Phase-0 wiring-proof placeholder inside the existing `ViewportPanel.tsx` with a real R3F
`<Canvas>` rendering the loaded mesh. **The existing Phase-0 chrome is preserved and wired live** — the
render-mode chips, camera-mode chips, stats overlay, and gizmo already exist and ship; Phase 2 makes
them functional and fills the canvas with real geometry. Do not re-skin the existing chrome; extend it.

### Canvas + chrome (preserve the inherited overlay layout)

- **3D canvas:** R3F `<Canvas>` filling the panel body, behind the existing overlay chips. Background:
  preserve the inherited radial gradient (`#2a2e26 → #141414`) + masked 28px dot grid as the neutral
  backdrop; the drei `<Grid>` (infinite floor grid) renders inside the scene on top of it.
- **Baseline viewport chrome (RESEARCH Discretion #3 — inspector default set):** infinite `<Grid>`,
  a **3-point lighting rig** (key/fill/rim) + neutral ambient, `<OrbitControls>` with damping,
  **camera auto-frame** to the loaded mesh bounds on open, a **wireframe toggle** (the existing "Wire"
  render-mode chip), a **bounding-box toggle**, and a **skeleton-helper toggle** (for VIEW-03 preview).
- **Render-mode chips (top-left, inherited):** `Solid | Wire | Textured` — already implemented as chips;
  wire them to the material/wireframe state. `Textured` is the default once a material resolves.
- **Camera-mode chips (top-right, inherited):** `⟲ Orbit | ✥ Pan | ⛶ Frame` — already implemented;
  `Frame` triggers camera auto-frame to mesh bounds. Each glyph-only chip keeps its inherited
  `aria-label` + `title` (Rule 5).
- **Stats overlay (bottom-left, inherited, extended):** mono `--text-xs`, `--color-text-faint`, extend
  the existing `persp · {W}×{H} · {fps} fps · SAB ✓` line to:
  `persp · {W}×{H} · {fps} fps · {verts} v · {tris} t · {draws} dc · zero-copy ✓`.
  fps must be live (`useFrame` tick). The `zero-copy ✓` token is the CORE-06 proof surfaced.
- **Gizmo (bottom-right, inherited):** 48×48 axis SVG (Y green / X red / Z blue), opacity 0.8 — keep as-is.

### States

| State | Content |
|-------|---------|
| Empty (no asset loaded) | Preserve the inherited centered placeholder pattern, recopied for Phase 2: `Viewport` (`--text-md`, `--color-text-muted`) + `Open a mesh from the Assets panel` (`--text-sm`, `--color-text-faint`) + `.sat / .apt to compose · .mgn / .msh to inspect` (`--text-xs`, `--color-text-faint`). |
| Loading / parsing (async, off-thread) | Indeterminate progress affordance (reuse `shared/AsyncProgress` from Phase 1) over the canvas: `Loading {filename}…` — the canvas stays interactive, status bar keeps ticking (CORE-06 responsiveness proof). |
| Rendered — composed (`.sat`/`.apt`) | Full composed appearance renders; the Appearance panel (Surface 6) shows the resolved graph; any missing deps surface a warning + placeholder, never a hard fail (D-04). |
| Rendered — leaf (`.mgn`/`.msh`) | Standalone mesh in bind pose; manual-attach affordance available to add a skeleton / animation (D-03). |
| Render error | Canvas shows `✕ Could not render {filename}` (`--color-danger`, `--text-md`) + reason + `Open in Hex` action (fall back to the Phase-1 hex inspector on the raw bytes). Never a blank canvas with no explanation. |

## Surface 2 — LOD Picker (RIGHT / Inspector — "Detail" section, or viewport overlay)

D-02: parse ALL LOD levels and let the user select one. LOD is an **external file graph** (`.lmg`/MLOD +
`.ldt`/LDTB) resolved by the appearance resolver — the picker lists the levels the resolver found, not
levels embedded in the mesh.

- **Placement:** a compact LOD control. Default home is the Inspector "Detail" section; MAY also surface
  as a small viewport-overlay dropdown (bottom-center) for fast switching. Pick one primary home; do not
  duplicate the control's state.
- **Anatomy:** one option per resolved LOD level, highest-detail (`LOD 0`) first:
  - level label `LOD {n}` (`--text-sm`),
  - distance range from `.ldt` (mono `--text-xs`, `--color-text-faint`, e.g. `0–8m`),
  - tri count for that level (mono `--text-xs`, right-aligned).
- **Selected state:** 2px accent left border + `--color-accent-dim` bg (inherited selection pattern);
  selecting re-binds the viewport geometry to that level's `BufferGeometry`.
- The client caps usable levels at `min(4, level_count)` (RESEARCH Pitfall 1) — show all parsed levels but
  note which are within the client's usable cap.

### States

| State | Content |
|-------|---------|
| Single LOD only | Collapse to a static `LOD 0` label (no picker) — `1 level` caption (mono, faint). |
| No LOD graph resolved (leaf mesh opened standalone) | `No LOD graph` (`--color-text-muted`) — the standalone mesh is its own single level. |

## Surface 3 — Customization Panel (RIGHT / Inspector — "Customization" section)

D-06 — the explicit "beats TRE Explorer" wow moment. Exposes the asset's shader customization variables
as **live color/palette pickers** that re-tint the mesh in real time (zero-allocation uniform mutation).

- **Section head:** `Customization` (`--text-md`, weight 600). If the asset has no customizable variables,
  hide the section and show nothing (do not show an empty header).
- **Per-variable row:** one row per customization variable resolved from the shader (`CustomizableShaderTemplate`):
  - variable name (`--text-sm`, e.g. `/private/index_color_1`),
  - the resolved `.pal` palette name + current index (mono `--text-xs`, `--color-text-faint`),
  - a **palette/color picker** control.
- **Palette picker control:** a horizontal swatch strip of the palette's entries (each swatch 18×18px,
  `--radius-sm`, 3px gap). The current entry is marked with a 2px accent ring (selection signal — paired
  with the swatch position, never color alone since every swatch is a different color). Clicking a swatch
  sets the customization index → mutates the shader uniform live (no re-parse, no realloc — D-09 GC bar).
  - **Swatch tiles are exempt scene content** — they show the artist's palette colors and are NOT bound
    by the chrome 60/30/10 / accent-scarcity rules (see Color note).
  - Hovered swatch: subtle `--color-surface-2` cell background behind it + `title` showing the packed
    `#AARRGGBB` value (mono).
- **Live readout:** the active packed color value shown in mono `--text-xs` (`#AARRGGBB`) next to the strip.
- **Reset affordance:** a `Reset` text action per variable (or section-level) returns to the asset's
  authored default index. Non-destructive (view-only re-tint; nothing written to disk).

### States

| State | Content |
|-------|---------|
| Asset has customization variables | Render the per-variable rows + swatch pickers. |
| Asset has no customization | Section hidden entirely. |
| Palette missing (`.pal` unresolved) | `⚠ palette missing: {name} — showing default tint` (`--color-warn`); render the base texture without the palette multiply (D-04 partial render). |

## Surface 4 — Material / Texture Inspector (RIGHT / Inspector — "Material" section)

D-07 — full multi-map material parity surfaced read-only so the user can see what resolved. Not an editor.

- **Section head:** `Material` (`--text-md`, weight 600). One block per shader group (a `.mgn`/`.msh` may
  have several PSDT groups → several Three.js materials).
- **Per-slot row** (diffuse / normal / specular / environment / emissive — the `.eft`/SSHT slots that
  resolved): slot name (`--text-sm`) · resolved texture filename (mono `--text-xs`, ellipsis) · format +
  dimensions (mono `--text-xs`, `--color-text-faint`, e.g. `DXT5 · 512×512 · 9 mips`) · a tiny thumbnail
  (optional, 18×18). Unresolved slots show `— none`.
- **Provenance / verification line:** the shader chain (`.sht` → `.eft`) + the SC-5 round-trip status for
  the parsed `.sht`/`.dds`/`.pal` (`byte-exact ✓` in accent, read-only — inherited from `01-UI-SPEC.md`).
- **DDS path note:** if a texture fell back to CPU decode (DXT2/4 or S3TC unavailable), surface the
  `≈ CPU-decoded` note (`--color-warn`) on that slot (RESEARCH Discretion #1 / Assumption A2).

### States

| State | Content |
|-------|---------|
| Material resolved | Per-slot rows + provenance line. |
| Diffuse-only / partial | Resolved slots shown; unresolved slots `— none`; no warning unless a referenced map is actually missing. |
| Texture missing (referenced but unresolved) | `⚠ missing: {texture} — magenta placeholder` (`--color-warn`); the viewport shows the magenta/checker placeholder on that slot (D-04). |

## Surface 5 — Animation Transport (BOTTOM / Data pane — new "Animation" tab, or viewport footer)

VIEW-03 / D-08 — full transport UX for skeletal playback. An `.ans` targets a **skeleton**, not a mesh,
so the picker browses animations from the mount and applies them to the loaded skeleton.

- **Placement:** a horizontal transport bar (30px tall, = `--tabstrip-h`) docked at the viewport footer
  OR as an "Animation" tab in the BOTTOM data pane. One primary home; do not duplicate transport state.
- **`.ans` picker:** a dropdown / searchable list of animations resolvable for the loaded skeleton
  (`--text-sm`); selecting one binds + plays it. Mono caption shows the variant tag (`CKAT 0001` /
  `KFAT 0003`).
- **Transport controls (left → right):**
  - `⏮ ▶/⏸ ⏭` play/pause + step (glyph-only buttons, each with `aria-label` + `title` — Rule 5),
  - **scrubber / timeline:** a draggable track (`--color-widget` track, `--color-accent` playhead fill),
    frame ticks; dragging scrubs to a frame,
  - **frame counter:** `{frame}/{total} · {t}s` (mono `--text-xs`),
  - **loop toggle** `↺` (chip-style active state — accent when on),
  - **speed control:** `0.25× / 0.5× / 1× / 2×` selector (mono `--text-xs`; `1×` default),
  - **skeleton-helper toggle** `⊹` (shows the bone helper in the viewport; chip-style active state).
- **GC-safe contract (D-09):** playback must not allocate per frame — the scrubber + counter read from
  reused state; the UI must not create objects in the `useFrame` loop (RESEARCH anti-pattern). This is a
  behavioral contract the executor honors, surfaced here so the checker knows the transport is the
  VIEW-03 "no per-frame GC hitching" surface.

### States

| State | Content |
|-------|---------|
| No skeleton loaded | Transport hidden / disabled with `Load a skinned mesh or skeleton to animate` (`--text-sm`, `--color-text-muted`). |
| Skeleton loaded, no animation selected | Transport enabled at frame 0; `.ans` picker prompts `Choose an animation…`; skeleton shown in bind pose. |
| Playing | Playhead advances; play button shows the pause glyph + accent pip. |
| Unsupported animation version (KFAT 0002) | `≈ unsupported legacy animation (KFAT 0002) — skipped` (`--color-warn`); do NOT mis-render — skeleton stays in bind pose (RESEARCH Discretion #4 / D-04 ethos). |

## Surface 6 — Appearance / Missing-Dependency Panel (RIGHT / Inspector — "Appearance" section)

D-03 / D-04 / D-05 — surfaces the cross-TRE resolution result and the missing-dependency warnings. The
viewer stays useful against an incomplete mount: render what resolved, substitute placeholders, warn.

- **Section head:** `Appearance` (`--text-md`, weight 600) — shows the opened entry + composition mode.
- **Open-mode indicator:** `composed (.sat)` vs `leaf (.mgn standalone)` (mono `--text-xs`, `--color-text-faint`)
  reflecting the D-03 smart-open path that was taken.
- **Resolved-graph readout:** a compact dependency list, each line `glyph · type · name`:
  - resolved → `✓ skeleton: appearance/skeleton/foo.skt` (accent `✓`),
  - resolved → `✓ mesh-gen: appearance/mesh/foo.mgn`,
  - resolved → `✓ shader: shader/foo.sht`,
  - missing → `⚠ texture: texture/bar.dds — placeholder` (`--color-warn`).
- **Missing-deps summary:** when ≥1 dependency is missing, a `--color-warn` banner heading
  `{n} missing dependencies` (`--text-md`) above the list; the viewport renders partial with placeholders
  (magenta/checker texture, default skeleton) — never a hard refusal (D-04, RESEARCH Pitfall 6).
- **Manual-attach affordance (leaf mode, D-03):** when a leaf `.mgn`/`.msh` is open standalone, an
  `Attach skeleton…` / `Attach animation…` action lets the user pick a `.skt` / `.ans` from the mount to
  compose manually (the "inspect this one mesh / debugging" flow).

### States

| State | Content |
|-------|---------|
| Fully resolved | All deps `✓ resolved`; no warning banner. |
| Partially resolved | `{n} missing dependencies` warn banner + per-dep status; placeholders in viewport. |
| Leaf standalone | `leaf (.mgn standalone)` mode + `Attach skeleton…` / `Attach animation…` actions. |

## Surface 7 — Extract & Export (Inspector header actions + export dialog)

VIEW-04 / D-10 — extract a raw asset; export the viewed (rigged + animated) mesh to glTF / COLLADA.
Export-only this phase (no re-import).

- **Extract action:** `Extract…` (verb + noun) in the loaded-asset header — writes the raw asset bytes
  from the mount to a user-chosen path (the SIE-successor "pull the file out" action). Non-destructive to
  the mount; never writes into reference installs (asset-safety discipline inherited from Phase 1 D-10).
- **Export action:** `Export…` opens a small dialog:
  - format radio: `glTF (.glb)` (default) | `COLLADA (.dae)` (mono `--text-sm`),
  - include-toggles: `Skeleton` (on if a skeleton is loaded), `Animation` (on if an `.ans` is playing),
  - current-LOD note: `exports LOD {n}` (mono `--text-xs`),
  - coordinate note (info): `Y-up · X-mirror applied` (`--color-info`, RESEARCH coordinate math),
  - `[Cancel] [Export]` buttons. `Export` is the primary action.
- **Export completion:** `✓ exported {file}` (`--color-accent`) in the status bar + a `Reveal in folder`
  text action. Export validity = "opens in an external tool" (D-10) — not a byte-exact gate.
- **Export-format dialog is the only modal surface in Phase 2** — introduce `--text-lg` (16px, weight 600)
  for the modal title here if a larger title is wanted (the Phase-0 note reserved 16px for the first modal;
  this is it). Otherwise the dialog title uses `--text-md` weight 600.

### States

| State | Content |
|-------|---------|
| Nothing loaded | `Extract…` / `Export…` disabled. |
| Loaded, exporting | Indeterminate progress in the dialog: `Exporting {format}…`; dialog buttons disabled. |
| Export failure | `✕ Export failed — {reason}` (`--color-danger`) in the dialog; `Retry` action; dialog stays open. |

## Accessibility Contract (inherited from `00-`/`01-UI-SPEC.md`, with Phase-2 additions)

All five Phase-0 rules + the Phase-1 additions apply unchanged. Phase-2 additions:

### Rule 1 (state never color-alone) — Phase-2 state combinations

| State | Signal combination |
|-------|--------------------|
| Render-mode / camera-mode chip active | accent fill **+** the chip is the only filled chip in its cluster (position) — inherited chip pattern |
| LOD level selected | 2px accent left border **+** `--color-accent-dim` bg **+** it is the highlighted row |
| Customization swatch selected | 2px accent ring **+** the swatch's grid position **+** mono `#AARRGGBB` readout — never color alone (every swatch is a different color, so the ring + readout carry the state) |
| Animation playing | pause glyph swap **+** accent play pip **+** advancing playhead — triple-encoded |
| Loop / skeleton-helper / bbox toggle on | accent chip fill **+** the glyph state **+** `title` reflecting on/off |
| Dependency resolved / missing | glyph (`✓`/`⚠`) **+** semantic color **+** mono caption — triple-encoded |
| Zero-copy / round-trip status | glyph (`✓`/`✕`) **+** color **+** mono caption — inherited triple-encoding |

### Rule 5 (icon-only controls need accessible name) — Phase-2 additions

Extend the Phase-0/1 exhaustive tables. Every glyph-only control declares BOTH `aria-label` and `title`.
The Phase-0 viewport chips (`⟲ Orbit`, `✥ Pan`, `⛶ Frame`, and the Solid/Wire/Textured chips) already
carry these — preserve them.

| Control | Glyph | `aria-label` / `title` |
|---------|-------|----------------------|
| Play / pause | `▶` / `⏸` | `"Play animation"` / `"Pause animation"` |
| Step back / forward one frame | `⏮` / `⏭` | `"Previous frame"` / `"Next frame"` |
| Loop toggle | `↺` | `"Toggle loop"` |
| Skeleton-helper toggle | `⊹` | `"Toggle skeleton helper"` |
| Bounding-box toggle | `⛶`-style box glyph | `"Toggle bounding box"` |
| Customization swatch | (color tile) | `"Set {variable} to palette index {n}"` (swatch carries a `title` of the `#AARRGGBB` value) |
| LOD option | (text "LOD n") | text-labelled — no extra aria needed beyond the visible label |
| Reset customization | (text "Reset") | `"Reset customization to default"` |
| Reveal exported file | (text) | `"Reveal exported file in folder"` |

### Rule 4 (WCAG AA contrast) — Phase-2 semantic tokens

- `--color-warn` (#e0a13a) on `--color-surface` (#242424): ≈ 6.8:1 — PASS AA (inherited; used heavily for missing-deps).
- `--color-info` (#4a8cff) on `--color-surface`: ≈ 4.9:1 — PASS AA (coordinate/format notes).
- For body warning/error copy, prefer `--color-text` with a semantic glyph + colored left border over
  semantic-colored body text below 4.5:1 (inherited Phase-1 rule).
- The 3D-canvas overlay chips sit on a translucent dark backdrop (`rgba(20,20,20,0.7)` + `blur(4px)`,
  inherited) so chip text retains contrast over arbitrary rendered scene content.

### Theme & keyboard

- All new surfaces re-style automatically via the inherited token system across all five themes incl.
  high-contrast (no per-component color overrides). The rendered **3D model** and **palette swatches** are
  scene content and are theme-independent by design.
- Every interactive element (chips, LOD options, swatch picker, transport controls, scrubber, export
  dialog controls, attach actions) is Tab-reachable with the inherited `--focus-ring` on `:focus-visible`.
  The scrubber is operable by keyboard (`←`/`→` step a frame, `Home`/`End` jump to ends).
