<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DDS bytes → WebGL GPU upload | Compressed DXT blocks uploaded directly; format enum must be validated against allowed set before upload |
| Palette bytes → uniform mutation | PaletteEntry r/g/b/a are uint8 (0–255); division to 0.0–1.0 float before set() |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-11 | Tampering | DDS GPU upload format selection | mitigate | Format enum validated against a whitelist {'DXT1','DXT3','DXT5'} before mapping to WEBGL extension constant; unsupported format (DXT2/DXT4/unknown) falls back to CPU decode path, never passes an arbitrary enum to the GPU driver |
| T-02-12 | Tampering | palette index OOB | mitigate | Before PaletteArgb lookup: assert 0 <= index < palette.entryCount; clamp to entryCount-1 if out-of-range (UI interaction can never pass an index past the swatch strip which is bounded by entryCount) |
| T-02-13 | Spoofing | ShaderMaterial uniform injection | accept | ShaderMaterial GLSL strings are authored source (not user input); uniforms are typed Vector4/Texture/float with no dynamic string construction — no injection surface. |
| T-02-14 | Tampering | DDS mip offset OOB | mitigate | parseDds (02-01) already validates sum of mip byteLengths <= (dataSize - 128); the renderer never accesses bytes beyond the validated mip table. |
</threat_model>
