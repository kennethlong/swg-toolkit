<ground_truth>
<!-- From 01-RESEARCH.md § TRE Override Resolution + Ground-Truth Reconciliation. DO NOT re-derive. -->
- A mount is an ordered list of search nodes sorted by priority, highest first; same-priority ties insert AFTER the last match (TreeFile.cpp:285-308, searchNodePriorityOrder = a.priority > b.priority).
- find(name) returns the FIRST node containing the file = highest-priority-wins / first-match-wins shadow (TreeFile.cpp:437-461).
- Within an archive: binary search over crc-sorted TOC, tie-broken by _stricmp on name (TreeFile_SearchNode.cpp:360-408). length==0 = tombstone that shadows lower archives.
- fixUpFileName (TreeFile.cpp:511-601): lowercase, backslash->forward slash, collapse repeated slashes, strip leading ./ ../.
- Search semantics (OUR design, not the client's): default case-insensitive substring over the flat name list; optional glob (*/?). NOT regex by default. Native returns matched indices (archives hold 100k+ entries — do not ship the full name list per keystroke).
- Async: Napi::AsyncWorker on the libuv pool for heavy mount/inflate; resolves a JS Promise zero-copy. Hold a Napi::Reference to input buffers across the async boundary (Pitfall 6).
</ground_truth>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| disk `.tre` -> native mount | Multiple untrusted archives mounted at once; malicious one must not corrupt resolution or crash the renderer (Path B) |
| native -> renderer (Path B) | A native parser crash takes the renderer (FND-01 residual); search over 100k entries must not DoS the UI thread |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06 | Denial of Service | search over 100k-entry name list per keystroke | mitigate | Native-side search returns matched INDICES only; debounce ~120ms; never ship the full name list to JS per keystroke (RESEARCH § Search) |
| T-01-07 | Tampering | override resolution with a hostile tombstone | mitigate | Tombstone (length==0) handling ported verbatim from TreeFile_SearchNode.cpp:397-401; resolve returns not-found, never an OOB read |
| T-01-08 | Denial of Service | multi-GB mount blocking the renderer (Path B) | mitigate | All heavy mount/inflate on Napi::AsyncWorker (libuv pool); status-bar tick is the responsiveness proof; no synchronous native call on the JS thread |
| T-01-09 | Info-disclosure | reference-install paths in the file picker | accept | UI only reads user-selected files; copy-to-scratch for tests is Plan-01 scope; the browser never writes to disk this phase |
</threat_model>
