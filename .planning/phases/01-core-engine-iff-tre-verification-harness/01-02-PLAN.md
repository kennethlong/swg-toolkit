<ground_truth>
<!-- From real client source + 01-RESEARCH § TRE Override Resolution. DO NOT re-derive. -->
- A mount is an ordered list of search nodes sorted by priority, highest first (`TreeFile.cpp:285-288`, `searchNodePriorityOrder => a.priority > b.priority`, inserted via `std::lower_bound` at :304).
- **Same-priority tie-break is a code-vs-comment AMBIGUITY:** the comment at `TreeFile.cpp:294-296` says new equal-priority nodes insert AFTER the last match, but `std::lower_bound` with the strict `priority_a > priority_b` predicate returns the FIRST non-`>` position, inserting the new node BEFORE existing equal-priority nodes. **Do NOT assert a direction in the spec — pin the actual CODE behavior with a test** (Task 1) and document which one the port reproduces.
- find(name) returns the FIRST node containing the file = highest-priority-wins / first-match-wins shadow (`TreeFile.cpp:437-461`).
- Within an archive: binary search over crc-sorted TOC, tie-broken by `_stricmp` on name (`TreeFile_SearchNode.cpp:360-408`). length==0 = tombstone that shadows lower archives (:397-401). The client stops at the FIRST tombstone.
- fixUpFileName (`TreeFile.cpp:511-601`): lowercase, backslash->forward slash, collapse repeated slashes, strip leading ./ ../.
- `resolveChain` is OUR algorithm (the client stops at the first tombstone and does not report the shadow chain). Define it explicitly: walk the priority list collecting every archive that contains `name`; the WINNER is the first non-tombstone (== `resolve(name)`); if the first containing archive is a tombstone the winner is represented as a tombstone (deleted) entry. Invariant: `resolveChain(name).winner === resolve(name)` for the non-tombstone case; for the tombstone-winner case the ShadowChain carries `tombstone: true` and a deleted-winner representation.
- Search semantics (OUR design): default case-insensitive substring over the flat name list; optional glob (*/?). NOT regex by default. Native returns matched INDICES (archives hold 100k+ entries — do not ship the full name list per keystroke).
- v6000 (NOT v0006) is enumerate-only/encrypted (`TreVersion.cs:79-86`); the warn chip + encrypted-entry UX target `6000` only.
- Async: Napi::AsyncWorker on the libuv pool for heavy mount/inflate; resolves a JS Promise zero-copy. Hold a Napi::Reference to input buffers across the async boundary (Pitfall 6).
</ground_truth>
