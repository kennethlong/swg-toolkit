<ground_truth>
<!-- From 01-RESEARCH.md § Block ordering on disk + Pitfall 5 + Ground-Truth Reconciliation. DO NOT re-derive. -->
- Builder write order (TreeFileBuilder.cpp:773-833): (1) 36-byte header with only token/version/numberOfFiles set; (2) file payloads in RESPONSE-FILE order (not crc/name order), each compressed-or-raw; (3) TOC (writeTableOfContents) entries in crc/name-sorted tocOrder, compressed if it helps; (4) name block null-terminated names in tocOrder; (5) MD5 block = numberOfFiles x 16 bytes, ALWAYS uncompressed; (6) seek back to 0, re-write the full header with the now-known toc/name offsets+sizes+compressors.
- MD5 block is written by the builder but NEVER read by SearchTree — a byte-identical repack MUST reproduce it + response-file payload ordering + the builder zlib level.
- zlib = level 6 (`78 9C`), used ONLY when strictly smaller and input > 1024 bytes (compressAndWrite tries only CT_zlib; emits 0 or 2).
- Recompressing untouched entries is NOT bit-stable: copy untouched entries' raw compressed slices verbatim; recompress only edited entries (TreWriter.cs:166-174).
- TOC = CRC-first 6-field records; stride 24 (0004/0005/5000) / 32 (0006/6000). Header = 36 bytes.
- v6000 payloads are encrypted -> NEVER written/round-tripped at the payload level; structural/enumeration only.
- CRC is computed over the fixUpFileName-normalized name (Crc::calculate).
</ground_truth>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| in-memory edits -> written `.tre` | The builder writes a new archive; a wrong layout produces a non-loadable archive (correctness, not external attack) |
| retail source archive -> repack | Untouched entry slices are copied verbatim; the source archive is read-only (copy-to-scratch, D-10) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-15 | Tampering | repack recompressing untouched entries (silent corruption / non-identity) | mitigate | Copy untouched raw compressed slices verbatim; recompress only edited entries (TreWriter.cs:166-174); the per-record slice-identity test is the regression guard |
| T-01-16 | Repudiation | byte-identity claim that isn't (missing MD5/ordering) | mitigate | Reproduce MD5 block + response-file payload order + zlib level 6; the self-built byte-identical test fails if any is wrong |
