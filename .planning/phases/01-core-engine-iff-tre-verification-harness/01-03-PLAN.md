<ground_truth>
<!-- From 01-RESEARCH.md § IFF Format + Ground-Truth Reconciliation + 01-PATTERNS.md. DO NOT re-derive. -->
- IFF is BIG-ENDIAN: tags + lengths byte-swapped via ntohl on read / htonl on write (Iff.cpp:522,539,637,643,713). 'FORM' reads as readable ASCII; length is MSB-first.
- Block framing: [4-byte Tag][4-byte BE uint32 length][payload]. Leaf length = payload bytes (NOT incl the 8-byte header).
- FORM container: 'FORM' · BE_u32(innerLen) · subTypeTag · children; innerLen INCLUDES the 4-byte sub-type tag (Iff.cpp:643 writes length+sizeof(Tag); :1144 reads getLength-sizeof(Tag)). FORM header = 12 bytes.
- Container discriminator: a block is a FORM iff first tag == TAG_FORM (Iff.cpp:1076-1095). Generic SIE-successor viewer ALSO recognizes LIST and `CAT ` (trailing space) as containers; PROP is a LEAF (OPEN-3 RESOLVED).
- NO PAD EVER (SWG no-pad quirk): EA-IFF-85 pads odd-length chunks; real SWG datatables DO NOT (IffReader.cs:307-327 detects, never requires; IffWriter.cs:141 emits none). Adding a pad fails byte-exact.
- Byte-exact serialize: hybrid-DOM verbatim re-emit — a clean node writes its captured source slice verbatim; only a dirty node reserializes (IffWriter.cs:98-187). Iff::write dumps the buffer verbatim (Iff.cpp:419-429).
- "Zero unexplained trailing bytes": surface any bytes after the last top-level block as an explicit trailing-bytes node (Iff.cpp:63-84).
- Security caps (Utinni IffReader.cs:174-195): per-chunk <= 64 MB; reject childEnd > parentEnd; reject non-0x20-0x7E FourCC bytes.
</ground_truth>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| arbitrary IFF file -> native parser | Third-party / malformed IFF bytes cross into the parser; a crash takes the renderer (Path B) |
| native ArrayBuffer -> HexInspector | Chunk bytes cross zero-copy; large chunks must not block or DoS the UI |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-10 | Tampering | IFF nested-chunk overflow (childEnd > parentEnd) | mitigate | Reject `declaredEnd > parentEnd` for nested chunks before recursing (IffReader.cs:185-195); malformed fixture proves clean rejection |
| T-01-11 | Tampering | Non-printable / garbage FourCC | mitigate | Reject tag bytes outside 0x20-0x7E (IffReader.cs:150-158) |
| T-01-12 | Denial of Service | Oversized chunk length (alloc attack) | mitigate | Per-chunk cap <= 64 MB; subtraction-form bound on declared length vs remaining buffer (IffReader.cs:174-195) |
| T-01-13 | Tampering | Byte-exact corruption via wrong pad handling | mitigate | Detect-don't-assume pad on read; emit NO pad on write; hybrid-DOM verbatim re-emit for unedited nodes — assertRoundTrip on odd-chunk-no-pad fixture is the regression guard |
| T-01-14 | Denial of Service | Multi-MB hex render blocking the UI | mitigate | Virtualized hex grid (only visible rows in DOM); large parse on Napi::AsyncWorker |
</threat_model>
