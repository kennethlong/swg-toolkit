NOTE the client enum is `CT_none=0, CT_deprecated=1, CT_zlib=2` and `isCompressed()` DEBUG_FATALs on code 1 (`TreeFile_SearchNode.h:166-172,211-214`) — so raw-deflate(1) handling is **Utinni-derived (TreFile.cs), NOT client-derived**: HANDLE code 1, do NOT fatal.
- Tombstone: a record with `length == 0` is deleted (shadows lower archives) — `TreeFile_SearchNode.cpp:397-401`.
- v6000 (Restoration) payloads are ENCRYPTED -> enumerate-only — applies to `6000` ONLY (NOT `0006`); `IsEnumerateOnly(v) => v == V6000` (`TreVersion.cs:79-86`). Byte-exact payload round-trip target = v0005.
- Name normalization (fixUpFileName, TreeFile.cpp:511-601): lowercase, backslash->forward slash, collapse repeated slashes, strip leading ./ ../. CRC = `Crc::calculate(normalizedName)` (`TreeFile_SearchNode.cpp:364`).
- Tie-break on equal CRC = `_stricmp` on the entry name (`TreeFile_SearchNode.cpp:382`); the binary search compares ONLY `mid`'s name, so an equal-CRC run needs a (crc,name) secondary sort + collision-safe scan (Task 3 collision fixture).
- Security caps (Utinni): per-record/block <= 256 MB; division-form count check `recordCount > Max/stride`; subtraction-form bound `offset > streamLen - len` (64-bit-widened); zlib output cap = `min(declared, Max)` with read-one-past-cap over-expansion detection.

- Net: even **v0005 field order is disputed** (client/tre_reader say crc-first; Utinni says size-first), and `0006` has THREE proposed layouts.
