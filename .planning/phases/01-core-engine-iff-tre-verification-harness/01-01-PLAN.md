<ground_truth>
<!-- Locked byte-layout facts from 01-PATTERNS.md / 01-RESEARCH.md Ground-Truth Reconciliation. DO NOT re-derive. -->
- TRE magic on disk = bytes `45 45 52 54` = "EERT". Version reads forward as ASCII "0004"/"0005"/"0006"/"5000"/"6000".
- Header = 36 bytes, `"<4s4s7I"`: token, version, numberOfFiles, tocOffset, tocCompressor, sizeOfTOC, blockCompressor, sizeOfNameBlock, uncompSizeOfNameBlock.
- TOC record = CRC-first for ALL versions (3 oracles agree): crc, length, offset, compressor, compressedLength, fileNameOffset. Stride = 24 bytes (0004/0005/5000), 32 bytes (0006/6000, +8 trailing pad).
- Compressor codes: 0=none, 1=raw deflate (handle — do NOT fatal like the C++ does), 2=zlib RFC1950 (0x78 header, body, 4-byte Adler). On-disk read length = `compressor==0 ? uncompressedSize : compressedLength`.
- Tombstone: a record with `length == 0` is deleted (shadows lower archives) — `TreeFile_SearchNode.cpp:397-401`.
- v6000 (Restoration) payloads are ENCRYPTED -> enumerate-only; byte-exact payload round-trip only on v0005.
- Name normalization (fixUpFileName, TreeFile.cpp:511-601): lowercase, backslash->forward slash, collapse repeated slashes, strip leading ./ ../.
- Security caps (Utinni): per-record/block <= 256 MB; division-form count check `recordCount > Max/stride`; subtraction-form bound `offset > streamLen - len`; zlib output cap = `min(declared, Max)`.
</ground_truth>
